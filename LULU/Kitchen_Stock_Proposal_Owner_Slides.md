# 🍱 ระบบจัดการครัว
## วัตถุดิบ · สั่งซื้อ · เบิก-โอน · ของเสีย
### สำหรับร้านอาหารหลายสาขา + ครัวกลาง

## ทำไมต้องมีระบบนี้

ปัจจุบัน ทุกอย่างยังอยู่ใน Excel + กระดาษ ทำให้...

<div class="grid2">

<div class="card">

### 😩 ปัญหาที่เจอบ่อย

- ราคาวัตถุดิบเปลี่ยน — สูตรไม่อัปเดต
- ไม่รู้ว่าจานไหน **ขาดทุน** จริง ๆ
- ของเสียเยอะ — แต่ไม่รู้แน่ชัดว่ากี่บาท
- เบิกของข้ามสาขาทาง LINE — หาย/ตกหล่น
- หาเอกสารใบสั่งซื้อย้อนหลังลำบาก
- ผู้จัดการสาขาทำงานซ้ำซ้อน

</div>

<div class="card">

### 🎯 สิ่งที่อยากได้

- รู้ **ต้นทุน/จาน** ทุกเมนูทันที
- รู้ **% ของเสีย** ของแต่ละวัน/เดือน
- เห็นสต๊อกของแต่ละสาขา **real-time**
- ใบสั่งซื้อพิมพ์ออกมาสวย ๆ ติดตามได้
- เบิกของข้ามสาขาเป็นระบบ มีหลักฐาน
- เปิดสาขาใหม่ **ตั้งระบบได้เร็ว**

</div>

</div>

## 1️⃣ คลังวัตถุดิบกลาง

ทุกอย่างอยู่ที่เดียว — **เปลี่ยนราคา 1 ที่ ทุกสูตรอัปเดตทันที**

| | เดิม (Excel) | ใหม่ (ในระบบ) |
|---|---|---|
| ราคา Bean Sprouts ขึ้นจาก ฿20 → ฿25 | ต้องไปแก้ทุกสูตรที่ใช้ | แก้ที่เดียว — Bibimbap, Japchae, Ramyeon ฯลฯ ปรับราคาตามอัตโนมัติ |
| Supplier ไหนขาย Beef ดีที่สุด | ต้องเปิดหลายไฟล์ | ดูในระบบหน้าเดียว — เทียบราคา/supplier ได้ |
| สาขาใหม่เปิด | copy Excel ทั้งกอง | ไม่ต้องทำอะไร — ใช้ฐานเดียวกันทันที |

> **ฐานข้อมูลตั้งต้นพร้อมแล้ว** — 79 วัตถุดิบ, 22 supplier, 60 สูตร ที่นำเข้าจาก Excel ของ LULU โดยตรง

## 2️⃣ สูตร & ต้นทุนต่อจาน

<div class="grid2">

<div>

**ตัวอย่าง: Beef Bibimbap (฿320)**

| วัตถุดิบ | ใช้ | ต้นทุน |
|---|---|---|
| Beef | 100 g | ฿220.00 |
| Rice | 200 g | ฿11.60 |
| Egg | 1 ชิ้น | ฿3.97 |
| ผัก + ซอส | — | ฿38.50 |
| **รวม × 1.30 (เผื่อเสีย)** | | **฿356.20** |

**กำไรขั้นต้น = ฿320 − ฿356 = ติดลบ! 🚨**

</div>

<div class="card">

### ระบบช่วยให้เห็น

- **ต้นทุน/จาน** ทุกเมนู update real-time
- **% ต้นทุน** vs ราคาขาย — เมนูไหนต่ำไป
- เมนูไหน **ขาดทุน** ขึ้น highlight แดง
- ปรับราคา/สูตร แล้วดูผลกระทบทันที
- เก็บ history — ราคาขึ้นกี่ %, กำไรเปลี่ยนแค่ไหน

</div>

</div>

> ปัจจุบัน LULU บวกเผื่อเสีย 30% ทุกจาน — ตัวเลขนี้ตั้งแก้ได้ตามจริง

## 3️⃣ สั่งซื้อ (Purchase Order)

ผู้จัดการเลือก supplier → กดเลือกของ → ระบบคำนวณราคา → พิมพ์ใบสั่งซื้อ

```
                  ผู้จัดการ
                     │
              สร้างใบสั่งซื้อ
                     │
              เจ้าของ/หัวหน้า อนุมัติ
                     │
            ┌────────┴────────┐
            ▼                 ▼
      🖨️ พิมพ์ใบ PO       📱 ส่งให้ supplier
      ส่ง supplier         ผ่าน LINE/Email
            │
       ของมาถึง — ผู้จัดการติ๊ก ✓ ของที่ได้รับครบ
            │
       ระบบเพิ่มสต๊อกอัตโนมัติ + บันทึกค่าใช้จ่าย
```

> 📄 **ใบ PO printable เหมือนเดิมที่ LULU ใช้อยู่** — มี checkbox / รายการ / qty / ราคา / ผู้สั่ง / ผู้อนุมัติ

## 4️⃣ ครัวกลาง — เบิกข้ามสาขา

<div class="grid2">

<div>

**ใช้งานจริง: สาขา A ขาดวัตถุดิบ**

```
[สาขา A]                  [ครัวกลาง]
   │                          │
   │ "ขอเบิก Kimchi 5 กก."     │
   │  Beef 2 กก., Egg 50 ฟอง   │
   ├─────────────────────────▶│
   │                          │ ตรวจ + อนุมัติ
   │                          │ (อาจปรับ qty)
   │                          │
   │◀──────  จัดส่ง ──────────┤
   │                          │
   ✅ สต๊อกสาขา A ↑           ✅ สต๊อกครัวกลาง ↓
```

</div>

<div class="card">

### ดีกว่าเดิม

- **มีหลักฐานทุกครั้ง** — ใครเบิก ใครอนุมัติ
- **อนุมัติผ่าน LINE** ได้ ไม่ต้องเปิดคอม
- **ติดตามได้** — ของถึงไหน ครบไหม
- **รายงาน**: สาขาไหนเบิกบ่อย, อะไรเบิกเยอะ
- **ครัวกลางตามทัน** — สั่ง supplier ก่อนของหมด

</div>

</div>

## 5️⃣ บันทึกของเสีย

<div class="grid2">

<div>

**หัวหน้าครัวบันทึก:**
- วัน/เวลา
- รายการ — เลือกจากวัตถุดิบ
- จำนวน (กรัม / ชิ้น)
- เหตุผล: หมดอายุ / ไหม้ / ตกหล่น / เตรียมเกิน / อื่น ๆ
- (ถ่ายรูปได้)

**ระบบทำให้:**
- คำนวณ **เป็นบาท** ทุกชิ้นอัตโนมัติ
- หักจากสต๊อกทันที
- รายงานรายเดือน — เสียกี่บาท % ของยอดซื้อ

</div>

<div class="card">

### Insight ที่จะได้

- เดือนนี้ของเสียกี่ %
- สาขาไหนเสียเยอะที่สุด
- วัตถุดิบไหนเสียบ่อย
- เหตุผลไหนซ้ำมากที่สุด
- ลด waste ลง 5% = เซฟกี่บาท/ปี

> ปัจจุบัน LULU มีบันทึกของเสีย 20 เดือน ใน Excel — นำเข้าระบบใหม่ได้

</div>

</div>

## ตัวอย่างวันทำงานจริง

<div class="grid2">

<div>

**🌅 เช้า — คุณป้อม (ผจก. สาขา 24BLVD)**
- เปิดแอป → ดูสต๊อก → Bean Sprouts ใกล้หมด
- กด **"เบิกจากครัวกลาง"** ขอ 10 กก.
- LINE แจ้งเตือนครัวกลางทันที

**🕙 สาย — เจ้าของ**
- ได้ LINE คำขอเบิก → กด **"อนุมัติ"** ในแอป
- ของถูกตัดจากครัวกลาง เพิ่มเข้าสาขา

**🍳 บ่าย — เชฟ Earth (LULU)**
- ทำกิมจิ batch ใหญ่ → ระบบหักวัตถุดิบตามสูตร
- พบเห็ดที่หมดอายุ → กด **"บันทึกของเสีย"** + ถ่ายรูป

</div>

<div>

**🌆 เย็น — เจ้าของ**
- ดูแดชบอร์ด:
  - ค่าวัตถุดิบเดือนนี้: **฿185,000**
  - ของเสียเดือนนี้: **฿8,200 (4.4%)**
  - จานกำไรน้อยสุด: Squid Bibimbap **2.1%**
- ตัดสินใจ: ปรับราคา / เปลี่ยนสูตร / เปลี่ยน supplier

**📞 พรุ่งนี้ — supplier**
- รับใบ PO ที่พิมพ์ออกมา (หรือผ่าน LINE)
- ส่งของ → ผจก. ติ๊กรับครบ → สต๊อกอัปเดต

</div>

</div>
