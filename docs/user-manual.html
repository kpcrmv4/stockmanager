1. แนะนำระบบ StockManager

ภาพรวมระบบและความสามารถหลัก

StockManager คืออะไร?

StockManager เป็นระบบจัดการคลังสินค้าและฝากเหล้าครบวงจร สำหรับร้านอาหาร/บาร์ที่มีหลายสาขา รองรับการทำงานตั้งแต่:

  • ฝาก/เบิกเหล้า — จัดการขวดเหล้าที่ลูกค้าฝากไว้ พร้อมวันหมดอายุ
  • เช็คสต๊อก — นับสต๊อกรายวัน เทียบกับข้อมูล POS อัตโนมัติ
  • แชทภายใน — สื่อสารภายในทีมพร้อม Action Card สำหรับรับงาน
  • โอน/ยืมสินค้า — โอนสต๊อกระหว่างสาขาและยืมสินค้า
  • วิเคราะห์ประสิทธิภาพ — Dashboard วัดผลพนักงาน ลูกค้า สาขา
  • แจ้งเตือนอัตโนมัติ — ผ่าน LINE และ Web Push Notification

Layout ของระบบ

ระบบมี 2 รูปแบบการแสดงผลตามบทบาท:

🖥 Desktop Layout (Owner, Accountant, Manager, HQ)

  • Sidebar ด้านซ้าย — เมนูหลักแบ่งเป็นหมวดหมู่ ยุบ/ขยายได้
  • Top Bar ด้านบน — ตัวเลือกสาขา, แจ้งเตือน, โปรไฟล์
  • พื้นที่หลัก — เนื้อหาหน้าจอหลัก

2. บทบาทผู้ใช้งาน (Roles)

อธิบายบทบาทและสิทธิ์การเข้าถึง
Role ชื่อภาษาไทย Layout หน้าแรกเมื่อ Login สิทธิ์หลัก
Owner เจ้าของร้าน Desktop /overview ทุกสิทธิ์ (รวมจัดการผู้ใช้, ตั้งค่า)
Manager ผู้จัดการ Desktop /store-overview สต๊อก, โอน, ยืม, รายงาน
Accountant บัญชี Desktop /reports ดูรายงาน
HQ คลังกลาง Desktop /hq-warehouse โอนสต๊อก, ดูรายงาน
Bar หัวหน้าบาร์ Mobile /bar-approval อนุมัติฝาก, เช็คสต๊อก
Staff พนักงาน Mobile /my-tasks ฝาก/เบิก, เช็คสต๊อก, ยืม
Customer ลูกค้า LINE LIFF /customer ดูเหล้าที่ฝาก, ขอเบิก
💡 หมายเหตุ: Owner และ HQ สามารถเข้าถึงข้อมูลทุกสาขาได้ ส่วน Manager, Bar, Staff จะเห็นเฉพาะสาขาที่ตนเองสังกัด

3. การเข้าสู่ระบบ

ขั้นตอนการ Login และ Register

Login สำหรับพนักงาน

1

เปิดเว็บไซต์ระบบ

เข้าเว็บไซต์ StockManager จะแสดงหน้า Login อัตโนมัติ

2

กรอก Username และ Password

กรอกชื่อผู้ใช้ (username) และรหัสผ่าน จากนั้นกดปุ่ม "เข้าสู่ระบบ"

3

ระบบนำไปหน้าแรกตาม Role

เมื่อ Login สำเร็จ ระบบจะพาไปยังหน้าแรกของ Role ตัวเองอัตโนมัติ

🔒
img-04-login-page.png
หน้า Login ของระบบ (Light/Dark Mode)

Login สำหรับลูกค้า (Customer)

ลูกค้าไม่ต้อง Login ด้วย username/password แต่เข้าผ่าน:

  • LINE LIFF — กดลิงก์จาก LINE Official Account ระบบจะยืนยันตัวตนอัตโนมัติ
  • Token URL — ได้รับ Link พิเศษจากพนักงาน
↑ กลับสารบัญ

📊 หน้า Overview (หน้าแรก Owner)

Dashboard ภาพรวมแสดง:

  • Summary Cards — จำนวนสาขา, ยอดฝากรวม, งานรออนุมัติ, แจ้งเตือนสต๊อก พร้อม Trend %
  • สถานะแต่ละสาขา — การ์ดแยกแต่ละสาขาแสดงงานค้าง/ปัญหา (รออนุมัติสต๊อก, รอชี้แจง, รอเบิก, ใกล้หมดอายุ, รอโอน) พร้อมลิงก์ไปหน้าที่เกี่ยวข้อง เรียงสาขาที่มีปัญหามากสุดไว้บนสุด
  • การ์ด HQ (คลังกลาง) — แสดงไอคอน Warehouse แทน Store, แสดงเฉพาะ "รอรับโอนจากสาขา" (ไม่แสดงข้อมูลฝาก/สต๊อกที่ไม่เกี่ยวข้อง)
  • Module Cards — ทางลัดไปยังระบบหลัก (สต๊อก, ฝาก, โอน, ยืม, รายงาน, ตั้งค่า)
  • Recent Activity — กิจกรรมล่าสุดของระบบ

1

เปิดลิงก์จาก LINE

2

ยืนยันตัวตนอัตโนมัติ

3

ดูเหล้าที่ฝากไว้

11. ระบบฝาก/เบิกเหล้า

จัดการขวดเหล้าที่ลูกค้าฝากไว้
Owner Bar Staff

หน้าหลักฝาก/เบิก

หน้าจอหลักแสดง:

  • Summary Cards — ฝากอยู่ในร้าน, รอยืนยัน, หมดอายุ, คำขอเบิก
  • Tabs — ทั้งหมด | ในร้าน | รอยืนยัน | หมดอายุ | รอนำส่ง HQ | VIP (แต่ละแท็บมี badge ตัวเลข)
  • Batch Transfer — ในแท็บ "หมดอายุ" มี checkbox เลือกหลายรายการ/เลือกทั้งหมด + ปุ่ม "โอนคลังกลาง (N)" สำหรับส่งรายการหมดอายุไป HQ แบบ batch
  • ค้นหา — ค้นหาด้วยรหัส/ชื่อลูกค้า/ชื่อสินค้า
  • ตัวกรองวันที่ — เลือกช่วงวันที่ (default: เมื่อวาน-วันนี้)
🍷
img-26-deposit-main.png
หน้าหลักฝาก/เบิก แสดง Summary + Tab Filter + Table/Cards

📋 รายละเอียดรายการฝาก (Deposit Detail)

กดที่รายการฝากเพื่อเข้าดูรายละเอียด จะเห็น:

สถานะการฝาก (Timeline)

แสดง 2 เส้นทางหลัก:

  • เส้นทางปกติ: รอยืนยัน → อยู่ในร้าน → รอเบิก → เบิกแล้ว
  • เส้นทางหมดอายุ: หมดอายุ → รอนำส่ง HQ → โอนคลังกลางแล้ว

การ์ด "การดำเนินการ" (ปุ่มแสดงตามสถานะ)

สถานะปุ่มที่แสดง
รอยืนยัน ยืนยันรับเข้าระบบ, ปฏิเสธรับฝาก (ต้องระบุเหตุผล), เปลี่ยน VIP, ทำเครื่องหมายหมดอายุ
อยู่ในร้าน เบิกเหล้า, เปลี่ยน VIP, ขยายวันหมดอายุ, ทำเครื่องหมายหมดอายุ
หมดอายุ โอนคลังกลาง (บังคับถ่ายรูป, ส่ง Action Card ไป HQ chat)

Flow โอนคลังกลาง (จากรายละเอียดรายการ)

1

กด "โอนคลังกลาง"

เปิด Modal แสดงข้อมูลสินค้า (ชื่อ, ลูกค้า, คงเหลือ, รหัสฝาก)

2

ถ่ายรูปสินค้า (บังคับ)

ต้องถ่ายรูปก่อนจึงจะกดยืนยันได้ + ใส่หมายเหตุ (ไม่บังคับ)

3

กด "ยืนยันโอนคลังกลาง"

ระบบสร้าง Transfer record, เปลี่ยนสถานะเป็น "รอนำส่ง HQ", ส่งข้อความแจ้งเตือนในแชทสาขา + ส่ง Transfer Action Card ไปแชท HQ

4

HQ ยืนยันรับ หรือ ปฏิเสธ

เมื่อ HQ รับ → สถานะเปลี่ยนเป็น "โอนคลังกลางแล้ว" — ถ้า HQ ปฏิเสธ → กลับเป็น "หมดอายุ"

🔄 ขั้นตอนการฝากเหล้า (Full Flow)

ขั้นตอนผู้ดำเนินการรายละเอียด
1. ลูกค้าขอฝากCustomerกรอกฟอร์มผ่าน LINE LIFF หรือพนักงานบันทึกให้
2. รับคำขอStaffดูในหน้า Deposit Requests แล้วกด "อนุมัติ"
3. Bar ยืนยันBarยืนยันรับสินค้าจริง + ถ่ายรูป ในหน้า Bar Approval
4. สถานะ "ในร้าน"ระบบBot แจ้งในแชท + LINE ลูกค้า
5. ลูกค้าขอเบิกCustomerกด "ขอเบิก" ผ่าน LINE LIFF
6. พนักงานจัดการStaffหยิบของ + ถ่ายรูปยืนยัน + กด "เบิกสำเร็จ"
7. แจ้งลูกค้าระบบLINE แจ้งลูกค้าว่าเบิกเสร็จแล้ว
🔄

16. ระบบแจ้งเตือน

แจ้งเตือนหลายช่องทาง

ช่องทางแจ้งเตือน

ช่องทางรายละเอียดผู้รับ
In-Appแจ้งเตือนในระบบ (กดกระดิ่ง)พนักงานทุกคน
PWA PushWeb Push Notification บนมือถือ/Desktopพนักงาน (ต้องเปิดใช้)
LINE Pushแจ้งเตือนผ่าน LINEลูกค้า + พนักงาน
LINE กลุ่มแจ้งเตือนในกลุ่ม LINE พนักงานสาขา (ปิดเป็น default)
Chat BotBot ส่ง Action Card ในแชทสาขาพนักงานในสาขา

ประเภทแจ้งเตือน

  • 🍷 ฝากเหล้าใหม่ — แจ้งพนักงานเมื่อมีลูกค้าฝาก
  • ✅ ยืนยันฝากแล้ว — แจ้งลูกค้าเมื่อ Bar รับสินค้า
  • 📥 คำขอเบิก — แจ้งพนักงานเมื่อลูกค้าขอเบิก
  • 📦 เบิกสำเร็จ — แจ้งลูกค้าเมื่อเบิกเสร็จ
  • ⚠ ใกล้หมดอายุ — แจ้งลูกค้าก่อนเหล้าหมดอายุ
  • 📋 ผลสต๊อก — แจ้ง Manager เมื่อเปรียบเทียบเสร็จ
  • 📜 รออนุมัติ — แจ้ง Owner เมื่อมีรายการรออนุมัติ
  • 📣 โปรโมชั่น — แจ้งลูกค้าโปรโมชั่นใหม่

17. ตั้งค่าระบบ

จัดการสาขา LINE การแจ้งเตือน
Owner

หน้าตั้งค่าหลัก

  • รายชื่อสาขา — ดูสาขาทั้งหมด พร้อมรหัสสาขา
  • เพิ่มสาขาใหม่ — กดปุ่ม + สร้างสาขา
  • Import Deposits — นำเข้าข้อมูลฝากจาก CSV
⚙
img-40-settings-main.png
หน้าตั้งค่าหลัก แสดงรายชื่อสาขา + ปุ่มเพิ่มสาขา

ตั้งค่าสาขา (Per-Store)

กดที่ชื่อสาขาเพื่อตั้งค่ารายละเอียด:

  • ข้อมูลสาขา — ชื่อ, รหัส, สถานะ active
  • LINE Configuration — Channel Token, Channel ID, Channel Secret
  • กลุ่ม LINE แจ้งเตือน — ID กลุ่มสำหรับ: สต๊อก, ฝาก, บาร์
  • การแจ้งเตือนลูกค้า — เปิด/ปิดแจ้งเตือนหมดอายุ, วันก่อนหมดอายุ
  • รหัสลงทะเบียนพนักงาน — Code สำหรับพนักงานใหม่

18. ระบบพิมพ์ใบเสร็จ

Print Station สำหรับ Thermal Printer 80mm
Owner Manager

Print Station

ระบบพิมพ์ใบเสร็จแบบอัตโนมัติ:

  • รองรับ Thermal Printer 80mm — สำหรับใบฝากเหล้า
  • Print Queue — ระบบคิวพิมพ์อัตโนมัติ Real-time
  • Auto-print — พิมพ์อัตโนมัติเมื่อมีงานใหม่
  • PWA — ติดตั้งเป็น App บนเครื่องพิมพ์

ตั้งค่า Print Station (5 ขั้นตอน)

1

เลือกสาขา

เลือกสาขาที่จะติดตั้ง Printer

2

ติดตั้ง PWA

เพิ่มเว็บเป็น App บนเครื่อง

3

เชื่อมต่อ Printer

เชื่อมต่อกับเครื่องพิมพ์ผ่าน LAN

4

ทดสอบพิมพ์

พิมพ์ทดสอบเพื่อตรวจสอบ

5

พร้อมใช้งาน

ดาวน์โหลด startup.bat (Windows) สำหรับเปิดอัตโนมัติ
